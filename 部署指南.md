## 步骤 2：注册免费 Redis（Upstash）

1. 打开 https://upstash.com/ ，用 GitHub 账号登录
2. 点击 **Create Database**
3. 区域选择 **Global**（或者靠近你后端的区域，比如 Tokyo）
4. 创建好后，在数据库详情页面找到 **Redis REST API** 或 **Connect** 标签
5. 复制以下信息（后面要用）：
   - **REDIS_URL**：格式类似 `rediss://default:xxxxx@xxx.upstash.io:6379`
   - 或者分开的：
     - **Host**：`xxx.upstash.io`
     - **Port**：`6379`
     - **Password**：`xxxxx`

## 步骤 3：配置后端连接信息

### 3.1 修改 application.yml（TiDB 模式）

我已经把 `application.yml` 恢复成了环境变量模式。你需要在部署平台上设置以下环境变量：

| 环境变量 | 值 | 说明 |
|---|---|---|
| `DB_HOST` | `gateway01.ap-northeast-1.prod.aws.tidbcloud.com` | TiDB 主机 |
| `DB_PORT` | `4000` | TiDB 端口 |
| `DB_NAME` | `ds_system` | 数据库名 |
| `DB_USERNAME` | `3yTdc4QGqF36Nuo.root` | TiDB 用户名 |
| `DB_PASSWORD` | **你重置后的密码** | TiDB 密码 |
| `REDIS_HOST` | `xxx.upstash.io` | Upstash Redis 主机 |
| `REDIS_PORT` | `6379` | Upstash Redis 端口 |
| `REDIS_PASSWORD` | `xxx` | Upstash Redis 密码 |
| `JWT_SECRET` | `任意随机字符串` | 建议用随机生成的 base64 字符串 |

### 3.2 TiDB 连接 URL 格式

在 TiDB Cloud 中，"Public" 连接必须使用 SSL。Spring Boot 的 URL 格式：

```
jdbc:mysql://gateway01.ap-northeast-1.prod.aws.tidbcloud.com:4000/ds_system?useSSL=true&serverTimezone=Asia/Shanghai&characterEncoding=utf8&allowPublicKeyRetrieval=true&sslMode=REQUIRED
```

`sslMode=REQUIRED` 表示强制使用 SSL，但不验证 CA 证书（适合 Render 等 PaaS 平台，不需要上传证书文件）。

## 步骤 4：部署后端到 Render

1. 打开 https://render.com/ ，用 GitHub 账号登录
2. 点击 **New** → **Web Service**
3. 选择你的 GitHub 仓库 `ds-system`
4. 配置如下：
   - **Name**：`ds-system-backend`
   - **Region**：`Singapore`（离国内最近）
   - **Branch**：`main`
   - **Runtime**：`Docker`
   - **Root Directory**：`backend`（因为 Dockerfile 在 backend 目录下）
5. 在 **Environment Variables** 中添加上面表格的所有变量
6. 点击 **Create Web Service**

Render 会自动构建并部署。首次部署可能需要 5-10 分钟。

## 常见问题

**Q：TiDB 的 SQL Editor 执行报错？**
A：TiDB 的 AUTO_INCREMENT 行为和 MySQL 略有不同，但上面的 SQL 兼容。如果报错，注意检查表名是否正确。

**Q：Render 部署失败，提示 Dockerfile 找不到？**
A：确保在 Render 配置中设置了 **Root Directory** 为 `backend`。

**Q：后端启动后连不上 TiDB？**
A：检查环境变量 `DB_PASSWORD` 是否正确（重置后的密码）。TiDB 连接必须使用 SSL，确保 URL 中有 `sslMode=REQUIRED`。

**Q：后端连接 Redis 超时？**
A：Upstash 的 Redis 是 TLS 连接，端口 6379。在 Spring Boot 中配置 `rediss://` 协议（注意是 `rediss` 不是 `redis`）。
